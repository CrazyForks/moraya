# v0.26.0 — Template Sharing + MCP Presets + Workflow Menu & SEO Publish + Full File Tree

## 概述

本迭代包含四个功能模块：
1. **Prompt Template 导入/导出/管理** — 支持社区分享模板（JSON 文件交换），本地优先
2. **MCP Presets 扩充** — 新增 6 个预置 MCP 服务，支持环境变量配置
3. **Workflow 菜单迁移 + SEO 集成到 Publish** — 将 AI Workflow 从底部状态栏迁移到原生菜单 + 编辑器右键菜单；SEO 作为 Publish 流程的可选步骤
4. **完整文件树 + 图片预览 Tab** — Sidebar 文件树升级为完整文件系统视图，支持按文件类型分流打开，图片可在 Tab 中预览

## F1: Prompt Template Import/Export/Manage

### F1.1 需求描述

用户可以通过 JSON 文件导入/导出 AI Prompt 模板，实现社区分享。模板支持两级存储：全局（`~/.moraya/templates/`）和知识库级（`{kbRoot}/templates/`），知识库模板优先。

### F1.2 类型扩展

**文件**: `src/lib/services/ai/templates/types.ts`

扩展 `AITemplate` 类型，支持内联文本（自定义模板）和 i18n key（内置模板）并存：

```typescript
export type TemplateSource = 'builtin' | 'global' | 'kb';

// AITemplate 新增可选字段（nameKey/descKey 变为可选）：
interface AITemplate {
  // ... 现有字段 ...
  name?: string;           // 内联名称（自定义模板用，优先于 nameKey）
  description?: string;    // 内联描述（自定义模板用，优先于 descKey）
  inputHint?: string;      // 内联输入提示（自定义模板用）
  source?: TemplateSource; // 运行时字段，不持久化到 JSON
  sourcePath?: string;     // 运行时字段：磁盘文件路径
  version?: string;        // 模板版本号（导出 JSON 中）
  author?: string;         // 模板作者（导出 JSON 中）
}

// TemplateParam 新增 label?: string（内联替代 labelKey）
// TemplateParamOption 新增 label?: string（内联替代 labelKey）

// 导出/导入文件格式
export interface TemplateExportFile {
  version: '1.0';
  templates: ExportableTemplate[];  // 支持批量导入/导出
}
export type ExportableTemplate = Omit<AITemplate, 'source' | 'sourcePath'>;
```

**显示名称解析辅助函数**（放在 `registry.ts`）：

### F1.3 存储层

**新文件**: `src/lib/services/ai/templates/template-storage.ts`

**存储规则**：
- 全局模板目录：`~/.moraya/templates/`（已有 `$HOME/.moraya/**` scope 权限）
- 知识库模板目录：`{kbRoot}/templates/`
- 每个模板一个独立 JSON 文件，文件名 = `{template.id}.json`
- 加载优先级：KB > 全局 > 内置（ID 冲突时高优先级覆盖低优先级）

**核心函数**：

| 函数 | 说明 |
|------|------|
| `loadAllCustomTemplates()` | 启动时加载全局 + 知识库模板 |
| `saveTemplate(tpl, scope)` | 保存模板到指定范围（global/kb） |
| `deleteTemplate(id, scope)` | 删除指定范围的模板文件 |
| `importTemplatesFromFile()` | 通过 Tauri file dialog 导入 JSON 文件 |
| `exportTemplates(templates)` | 通过 Tauri save dialog 导出为 JSON |

**导入验证规则**：
- 必需字段检查：`id`, `category`, `icon`, `flow`, `contentSource`, `systemPrompt`, `userPromptTemplate`, `defaultActions`
- 必须有 `name` 或 `nameKey`
- `flow` / `contentSource` / `defaultActions` 枚举校验
- ID 冲突时提示用户选择覆盖或自动重命名

**复用现有基础设施**：`invoke('read_dir_recursive')`, `invoke('read_file')`, `invoke('write_file')`, `homeDir()`, `filesStore.getActiveKnowledgeBase()`, Tauri dialog plugin

## F4: 完整文件树 + 图片预览 Tab

### F4.1 需求描述

将 Sidebar 的 tree 视图从"知识库文档树"升级为"完整文件系统树"：显示所有文件夹和文件（包括非 .md 文件），按文件类型显示不同图标，保留目录标色，按文件类型分流打开行为，图片在 Tab 中预览。

### F4.2 视图模式

**现有两种模式保持**：
- `'tree'`（升级后）— 完整文件系统树，显示所有文件和文件夹（含扩展名），depth=3
- `'list'` — 知识库文档模式，仅显示 .md 文件的预览文本（现有行为不变）

视图模式已通过 `files-prefs.json` 持久化（`sidebarViewMode`），关闭窗口后下次打开恢复上次选择。

**模式切换时刷新文件树**：`toggleViewMode()` 调用 `refreshFileTree(folderPath)` 重新加载，tree 模式传 `allFiles: true`，list 模式传 `allFiles: false`。

### F4.7 文件类型分流（点击行为）

**文件**: `src/routes/+page.svelte` — `doFileSelect()`

所有文件点击统一走 `onFileSelect` → `doFileSelect()`，在其中按文件类型分流：

```typescript
async function doFileSelect(path: string, mySerial: number) {
  const fileName = getFileNameFromPath(path);

  if (isImageFile(fileName)) {
    // 图片：创建 image tab（只读预览）
    tabsStore.syncFromEditor();
    tabsStore.openFileTab(path, fileName, '', null, true, true);  // isImage=true
    return;
  }

  // 非图片：正常打开编辑器 tab（现有逻辑）
  tabsStore.syncFromEditor();
  const fileContent = await loadFile(path);
  tabsStore.openFileTab(path, fileName, fileContent, mtime, true);
}
```

**文件类型判断**：复用 `file-service.ts` 已有的 `isImageFile()` 函数（基于 `IMAGE_EXTENSIONS` Set），导出为 `export function`。不额外创建 `file-types.ts`。

**图片读取**：`readImageAsBlobUrl()` 改用 `invoke('read_file_binary')` 替代 Tauri fs plugin 的 `readFile()`，因为 Rust 命令的 `validate_path()` 允许整个用户 home 目录，scope 覆盖更广。
