# v0.28.0 — 批量上传图片至图床 + 新建文档自动命名

## 概述

本次迭代围绕 **编辑效率** 和 **资产管理** 两个方向增强：

1. **批量上传图片至图床** — 右键菜单一键上传文档中所有图片（Markdown `![](...)` 和 HTML `<img>` 标签），支持本地路径与远程 URL 两种图源，上传结果通过 AI 对话面板实时反馈
2. **新建文档自动命名** — 保存新文档时自动从 H1 标题或首行文本提取文件名，减少手动命名成本；支持 MORAYA.md 规则文件自定义命名模板

## F1: 批量上传图片至图床

### F1.1 功能入口

**右键菜单新增选项**：在视图模式下，非图片区域右键弹出的编辑器上下文菜单中新增 **"上传所有图片至图床"** 选项。

- 文档中至少存在一个图片标签时，选项可点击
- 文档中无图片时，选项灰色禁用（`disabled` 状态）
- 图片检测范围：Markdown `image` 节点 + `html_inline`（`<img>` 原子节点）+ `html_block`（含 `<img>` 标签的 HTML 块）

**文件**: `src/lib/editor/EditorContextMenu.svelte`

新增 props：
- `hasImages: boolean` — 控制菜单项是否可点击
- `onUploadAllImages: () => void` — 点击回调

### F1.2 图片收集与上传逻辑

**文件**: `src/lib/editor/Editor.svelte`

#### 图片类型识别

遍历 ProseMirror 文档树，收集三种图片节点的 `src`：

| 节点类型 | 取值方式 | 示例 |
|---------|---------|------|
| `image`（Markdown） | `node.attrs.src` | `![alt](./images/photo.png)` |
| `html_inline`（HTML 原子） | 从 `node.attrs.value` 正则提取 `src` | `<img src="..." width="50%">` |
| `html_block`（HTML 块） | 从 `node.textContent` 正则匹配所有 `<img>` | 多行 HTML 中的 `<img>` 标签 |

收集时按 URL 去重（`Set<string>`），避免同一图片重复上传。

#### 图源解析

`resolveImageBlob(src)` 统一处理所有图源类型：

| 图源 | 处理方式 |
|------|---------|
| `http://` / `https://` | 通过 `fetchImageAsBlob()` 经 Tauri HTTP 代理下载 |
| `blob:` / `data:` | 浏览器 `fetch()` 直接获取 |
| 本地路径（相对/绝对） | 根据当前文档路径解析绝对路径 → `readImageAsBlobUrl()` → `fetch()` |

#### 文档更新

上传完成后，使用 `oldSrc → newUrl` 映射表在**单个事务**中批量替换所有匹配 URL：

| 节点类型 | 替换方式 |
|---------|---------|
| `image` | `tr.setNodeMarkup(pos, undefined, { ...attrs, src: newUrl })` |
| `html_inline` | 更新 `value` 属性中的 `src` 值（保留原 HTML 标签结构） |
| `html_block` | 替换文本内容中的 URL 子串（保留原 HTML 结构） |

**关键设计**：HTML 标签的其他属性（`width`、`height`、`style`、`class` 等）完全保留，仅替换 `src` URL 部分。

### F1.3 AI 对话面板反馈

**替代 Toast 通知**：所有上传结果改为通过 AI 对话面板展示，提供更完整的操作记录。

#### 消息流程

1. **点击上传** → 立即弹出 AI 面板（若隐藏则强制打开），显示"正在上传图片至图床..."提示
2. **第一条结果到达** → 移除"正在上传"提示，显示具体结果
3. **每张图片** → 独立一条消息：成功（绿色）或失败（红色）
4. **全部完成** → 输出汇总消息："上传完成：X/Y 张成功"

#### 消息样式

扩展 `ChatMessage` 接口，新增 `isSuccess?: boolean` 字段：

| 字段 | 颜色 | 用途 |
|------|------|------|
| `isSuccess: true` | 绿色（`#f0fdf4` 底 + `#166534` 文字） | 上传成功 |
| `isError: true` | 红色（`#fef2f2` 底 + `#991b1b` 文字） | 上传失败 |
| 无标记 | 默认样式 | 汇总/提示消息 |

**文件**: `src/lib/components/ai/AIChatPanel.svelte` — 新增 `.notify-success` / `.notify-error` CSS 类

#### AI Store 扩展

**文件**: `src/lib/services/ai/types.ts`

- `ChatMessage.isSuccess?: boolean` — **新增**，用于绿色成功样式

## F2: 新建文档自动命名

### F2.1 启用条件

- 设置中 `autoIndexOnSave` 开启（即知识库本地索引功能已启用）
- 文档内容非空

未开启本地索引时，保存行为与现有完全一致（默认 `untitled.md`）。

### F2.2 Rust 后端关键词提取

**文件**: `src-tauri/src/commands/kb.rs` — 新增 `kb_suggest_filename` Tauri 命令

文件名由 Rust 端基于分词和 TF 评分生成，而非简单取第一行文字。

#### 命名策略（优先级从高到低）

| 优先级 | 策略 | 说明 | 示例 |
|--------|------|------|------|
| 1 | H1 标题 | 文档含 `# ...` 时直接使用 | `# API 设计指南` → `API-设计指南` |
| 2 | TF 关键词 | 无 H1 时，分词 → 停用词过滤 → TF 评分 → 取 Top-N 拼接 | 文章关于 Rust 和性能 → `rust-性能-优化` |
| 3 | 首行文本 | 以上均无结果时取首个非空行 | `这是一篇介绍...` → `这是一篇介绍` |

#### CJK 分词与双字组合

复用知识库索引的 `tokenize()` 分词器（CJK 单字 + 英文词组），并新增 **CJK 双字组合**（bigrams）：

- 原始分词：`["机", "器", "学", "习", "入", "门"]`（每个 CJK 字符为独立 token）
- 双字组合后：`["机器", "器学", "学习", "习入", "入门"]`
- TF 评分后选出：`["机器", "学习", "入门"]`

#### 停用词过滤

| 类型 | 示例 |
|------|------|
| CJK 停用字（59 个） | 的、了、在、是、我、有、和、就、不、人... |
| 英文停用词（95 个） | the, be, to, of, and, in, that, have, it... |
| 纯数字 | 123, 2026... |

#### TF 评分公式

```
score = tf × (1 / (1 + ln(tf)))
```

高频词被对数抑制，避免文档中大量重复的通用词主导文件名。

#### 文件名清洗与截断

- 特殊字符（标点、符号）替换为 `-`
- 连续 `-` 合并，首尾 `-` 去除
- 截断时优先在 `-` 处断开，避免截断中文词语
- 默认最大长度：50 字符（MORAYA.md 规则可覆盖，范围 10-120）

### F2.3 前端调用

**文件**: `src/lib/utils/filename-suggest.ts`（新增）

前端通过 `invoke('kb_suggest_filename', { content, maxLength })` 调用 Rust 后端。MORAYA.md 命名模板解析保留在前端（`parseNamingRules` + `applyTemplate`），将 Rust 返回的关键词文件名填入模板的 `{title}` 占位符。

### F2.4 MORAYA.md 命名规则

`parseNamingRules(morayaContent)` 从 MORAYA.md 中查找命名规则 section：

**识别规则 section**：标题含"命名"、"文件名"、"naming"、"filename"等关键词（`## 文件命名规则`、`### Naming Convention` 等）

**支持的指令**：

| 指令 | 格式 | 默认值 |
|------|------|--------|
| 格式/pattern | `格式：{date}-{title}` 或 `pattern: {title}` | `{title}` |
| 最大长度/max length | `最大长度：50` 或 `max length: 80` | `50` |

**模板占位符**：

| 占位符 | 说明 | 示例 |
|--------|------|------|
| `{title}` | 提取的标题文本 | `我的文章` |
| `{date}` | 当前日期 YYYY-MM-DD | `2026-04-01` |
| `{YYYY}` | 年份 | `2026` |
| `{MM}` | 月份（补零） | `04` |
| `{DD}` | 日期（补零） | `01` |

**MORAYA.md 中的命名规则示例**：

```markdown
## 文件命名规则

格式：{date}-{title}
最大长度：60
```

保存效果：`# API 设计指南` → `2026-04-01-API-设计指南.md`

### F2.5 保存流程集成

**文件**: `src/routes/+page.svelte`

新增 `computeSuggestedPath(content)` 函数：

```
Cmd+S（新文档）
├── autoIndexOnSave 关闭？ → defaultPath = 'untitled.md'（不变）
└── autoIndexOnSave 开启
    ├── 在知识库中 + MORAYA.md 有命名规则？
    │   → 按规则模板生成（如 2026-04-01-我的文章.md），保存到 KB 根目录
    ├── 在知识库中 + 无命名规则？
    │   → 提取 H1 标题作为文件名，保存到 KB 根目录
    └── 不在知识库中？
        → 提取 H1 标题作为文件名，保存到系统默认位置
```

**文件**: `src/lib/services/file-service.ts`

`saveFileAs(content, suggestedPath?)` 新增可选参数，传入保存对话框的 `defaultPath`。用户在对话框中仍可自行修改。

## 验证方案

### F1 批量上传图片

1. **菜单状态**：文档无图片 → 右键菜单"上传所有图片"灰色不可点击
2. **Markdown 图片**：文档含 `![](./local.png)` + `![](https://example.com/remote.png)` → 全部上传成功
3. **HTML 图片**：文档含 `<img src="./photo.jpg" width="50%">` → 上传成功，`width` 等属性保留，仅 `src` 被替换
4. **混合场景**：同一文档中 Markdown + HTML 图片混合 → 全部正确处理
5. **AI 面板通知**：面板隐藏时点击上传 → 面板自动弹出 → 显示"正在上传" → 每张结果独立显示（绿色/红色）→ 最后显示汇总
6. **未配置图床**：未设置图床时点击 → AI 面板弹出红色提示
7. **单张上传**：右键图片 → "Upload to Image Host" → AI 面板绿色成功/红色失败

### F2 自动命名

8. **未开启索引**：`autoIndexOnSave` 关闭 → 保存对话框默认 `untitled.md`（行为不变）
9. **开启索引 + H1 标题**：文档含 `# API 设计指南` → 保存对话框默认 `API-设计指南.md`
10. **开启索引 + 无标题**：文档只有正文 → 取首行文本作为文件名
11. **开启索引 + 空文档**：空内容 → 保存对话框默认 `untitled.md`
12. **MORAYA.md 命名规则**：规则 `格式：{date}-{title}` → 保存对话框默认 `2026-04-01-API-设计指南.md`
13. **在知识库中**：保存对话框路径默认指向 KB 根目录
14. **特殊字符处理**：标题含 `C++/C#：入门指南` → 文件名 `C-C-入门指南.md`
15. **用户可修改**：自动命名仅作为默认值，用户在保存对话框中可自由修改
